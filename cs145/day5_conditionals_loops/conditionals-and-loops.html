For sake of example, we will begin with the toy Hello.java program that we have been using thus far in class:

public class Hello {
	public static void main(String[] args) {
		System.out.println("Hello, World!");
		// More code written here...
	}
}

You can assume that much of the code written below our print statement in the main method.

Conditional Statements

Conditional statements are an essential part of any programming language, and Java is no exception. They allow us to control the flow of our program based on certain conditions. In Java, conditional statements are implemented using the if and else keywords.

The basic syntax for an if statement is as follows:

if (condition) {
	// code to execute if condition is true 
}

Here, condition is an expression that evaluates to either true or false. If condition is true, the code block indented under the if statement is executed. If condition is false, the code
block is skipped. As a note, our Boolean expression must be enclosed in paratheses.

For example, let's say we want to print a message if a certain variable x is greater than 10. We could use the following code:

int x = 11;

if (x > 10) {
	System.out.println("My Variable is greater than 10!");
}

This will display:

My Variable is greater than 10!

However, if x were less than or equal to 10, then the message would not display at all.

In addition to the if keyword, Python also provides the else keyword to execute code if the if condition is false. The basic syntax for an if-else statement is as follows: 

if (condition) {
	// code to execute if condition is true 
} else {
	// code to execute if condition is false
}

For example, let's say we want to print a message if a certain variable x is greater than 10, and a different message if it is not. We could use the following code:

int x = 1;

if (x > 10) {
	System.out.println("My Variable is greater than 10!");
} else {
	System.out.println("My Variable is not greater than 10!");
}

This will display:

My Variable is not greater than 10!

In Java, we can chain these else and if keywords together, which allows you to check multiple conditions in sequence. The program evaluates each condition from top to bottom and executes the code block for the first true condition. The basic syntax is as follows:

if (condition1) {
	// code to execute if condition1 is true
} else if (condition2) {
	// code to execute if condition2 is true
} else if (condition3) {
	// code to execute if condition3 is true
} else {
	// code to execute if all the conditions are false.
}

Suppose we want to print a different message when a variable x is positive, negative, or zero:

int x = 10;

if (x > 0) {
	System.out.println("My variable is positive!");
} else if (x == 0) {
	System.out.println("My variable is zero!");
} else {
	System.out.println("My variable is negative!");
}

One important note about code blocks in Java: when you define a variable, it is scoped to the block where you define it.

For example:

if (condition) {
	int x = 10;
}
System.out.println(x);

This code will result in a compilation error, as the variable x is scoped only to the if-block. We can fix this code like this:

int x;
if (condition) {
	x = 10;
}
System.out.println(x);

Now, x is defined first outside the if-block, and the value of x is reassigned if condition is true.
*Note: This still may cause an error if condition is false as x will remain uninitialized. So, it may be best to assign x some default value like 0.*

Ternary Operator

The ternary operator allows for a succinct way of writing conditional statements on one line. However, I generally do not recommend that you use the ternary operator as it could make your code harder to understand. However, you may see this out in the wild, so it is important that you understand how it works.

The basic syntax for a conditional statement using the ternary operator is as follows:

condition ? value_if_true : value_if_false;

The variable condition is similar to the Boolean expression you would place after the if keyword in an if-statement. If condition is true, then the value_if_true is returned. If condition is false, then the value_if_false is returned.

Let's try to print a message if a certain variable x is greater than 10, and a different message if it is not using the ternary operator:

int x = 1;

String message = (x > 10) ? "My Variable is greater than 10!" : "My Variable is not greater than 10!"

System.out.println(message);

Switch Statements

Let's say that we have a RPG where there are a number of different character classes a player can select. Perhaps we can write code like this to display their class information:

public class RPG {
	public static void main(String[] args) {
		String playerClass = args[0];
		if (playerClass.equalsIgnoreCase("rogue")) {
			System.out.println("You are a Rogue");
		} else if (playerClass.equalsIgnoreCase("wizard")) {
			System.out.println("You are a Wizard");
		} else if (playerClass.equalsIgnoreCase("Warrior")) {
			System.out.println("You are a Warrior");
		} else {
			System.out.println("You are a Bard");
		}
	}
}

While this is a valid way of accomplishing this task, an alternative would be to use a switch statement, which is a control structure that allows you to execute one block of code out of many options based on the value of an expression.

For example, this code accomplishes the same task with a switch statement:

public class RPG {
	public static void main(String[] args) {
		String playerClass = args[0];
		switch (playerClass.toLowerCase()) {
			case "rogue":
				System.out.println("You are a Rogue");
				break;
			case "wizard":
				System.out.println("You are a Wizard");
				break;
			case "warrior":
				System.out.println("You are a Warrior");
				break;
			default:
				System.out.println("You are a Bard");
		}
	}
}

For some, this code is considered to be far cleaner than the code above (I would agree).

*Note: You need the break in order to prevent fall-through behavior in Java. Without a break, our program will continue executing all subsequent case blocks until it encounters a break or the end of the switch statement, even if the case condition is not satisfied.*

While Loops

In Java, loops are used for executing a block of code repeatedly until a certain condition is met.

A while loop executes a block of code as long as the specified condition is true. The syntax for a while loop is as follows:

while (condition) {
	// code to execute while condition is true 
}

Let's look at an example of a while loop and break down its components: 

int counter = 0;
while (counter < 5) {
	System.out.prtintln("Hello, World!");
	counter++;
}

This will display:

Hello, World!
Hello, World!
Hello, World!
Hello, World!
Hello, World!

In this example, we have:

  1. Initialization: int counter = 0. Before entering the while loop, we initialize a variable named counter with a value of 0.
  2. Condition: counter < 5. The while loop will keep running as long as this condition remains true.
  3. Code block: The block of code in curly braces under the while statement. This is the code that will be executed repeatedly.
  4. Update: counter: counter++. Inside the while loop, we increment the counter variable by 1, ensuring that the loop will end when the condition becomes false.

Do-While Loops

In Java, we have another type of loop called a do-while loop. It is very similar to a while loop, but it guarantees that the body of our loop executes at least once.

The syntax of a do-while loop is as follows:

do {
	// code to execute while condition is true 
} while (condition)

In a normal while loop, the condition is checked before executing the code block. If the condition is false, the code block is never executed. In contrast, a do-while loop executes the code block first and then checks the condition to determine whether the loop should continue or stop.

Let's look at an example:

int counter = 0;
do {
	System.out.prtintln("Hello, World!");
	counter++;
} while (counter < 0)

This will display:

Hello, World!

Even though the condition of the while loop is false, the code block executes once before the loop stops.

Loop Control Statements

Loop control statements allow you to modify the behavior of a loop during its execution. The two main loop control statements are break (which we saw earlier with swich statements) and continue.

break: Terminates the loop and exits immediately.
continue: Skips the rest of the code block and starts the next iteration of the loop.

Here's an example that demonstrates the use of both break and continue in a while loop: 

int counter = 0;  
while (counter < 10) {
	counter++;
	if (counter % 2 == 0) {
		continue;
	}
	if (counter == 9) {
		break;
	}
	System.out.println(counter);
}

This will display:

1
3
5
7

For Loops

Another type of loop in Java is the for loop. This kind of loop does not give you any more power than the while loop, in fact they are equivalent, as any for loop can be written as a while loop, and any while loop can be written as a for loop. We use for loops because they offer us a syntactically neat way of iterating with some counter variable.

The syntax of a for loop is as follows:

for (int i = 0; i < 10; i++) {
	// code to execute while i is less than 10.
}

Let's look at an example of a for loop and break down its components: 

for (int counter = 0; counter < 5; counter++) {
	System.out.prtintln("Hello, World!");
}

This will display:

Hello, World!
Hello, World!
Hello, World!
Hello, World!
Hello, World!

In this example, we have:

  1. Initialization: int counter = 0. In the loop header of the for loop, we initialize a variable named counter with a value of 0.
  2. Condition: counter < 5. The for loop will keep running as long as this condition remains true.
  3. Code block: The block of code in curly braces under the loop header. This is the code that will be executed repeatedly.
  4. Update: counter: counter++. Inside the loop header, we increment the counter variable by 1, ensuring that the loop will end when the condition becomes false.

Notice that this for loop is functionally identical to the while loop that we made earlier. All the primary components are the same, but the syntax is different.
