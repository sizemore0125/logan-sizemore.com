Variables

In programming, a variable is a named storage location that can hold a value. Variables are
used to store data that can be manipulated and used in different parts of a program. In
Java, variables are created by assigning a value to a name AND a type.

Initializing a Variable

In Java, you can create a variable by assigning a value to a name and a type using the = operator.
The general syntax for creating a variable is as follows:

int myVar = 29;

In this case, int specifies the type of the variable. Following the type, myVar serves as the name or identifier for the variable. The = is the assignment operator, and 29 is the value assigned to the variable.

Java is a Strongly Typed language, meaning that every variable has a specific type, and all assignments must adhere to strict type compatibility rules.

The name of a variable in Java can include letters, numbers, underscores (_), and dollar signs ($), but it cannot begin with a number or contain whitespace. Also, variables cannot be any of Java's reserved keywords (e.g., int, class, public, etc.)

Stylistically, we use camel-case notation for variables and methods (functions) in Java. This means that the first letter of the variable name is lowercase, and every subsequent word starts with an uppercase letter.

When the Java Virtual Machine executes this code, it will allocate some memory, then it will store the number 29 in that memory, and finally, it will attach myVar to that memory location.

Metaphor: We have a box, we put 29 in the box, and myVar is the label we put on that box.

Reassigning Variables

In Java, you can reinitialize variables once you have initialized them.

For example:

int myVar = 29;
myVar = 49;

Notice that we did not specify a type when we reassigned the value of this variable. In fact, you cannot change the type of a Java variable once it has been initialized.

For example, if we tried:

int myVar = 29;
int myVar = 35;

It would result in this error:

Hello.java:4: error: variable myVar is already defined in method main(String[])
                int myVar = 35;
                    ^
1 error

However, we do not need to assign a value to a variable at all. We can simply declare a variable with a type and an identifier like this:

int myVar;

Then, later in the program, we can give that variable a value:

myVar = 10;

You may be wondering what happens when we try to display an uninitialized variable. In this case, if we try to print myVar without initializing it, we get:

Hello.java:4: error: variable myVar might not have been initialized
                System.out.println(myVar);
                                   ^
1 error

We can also assign the value of one variable to another. For example, if we wanted to declare the variable myVar2 and give it myVar, we can simply do this:

int myVar = 29;
int myVar2 = myVar;
System.out.println(myVar2);

This code will then display:

29

*Note: For the examples above I used the identifier myVar, which is a bad variable name. When creating variables in Java, it's important to choose a descriptive and meaningful
name that reflects the purpose of the variable.*

Java Data Types

There are two categories of data types in Java: primitive types for storing values and reference types for storing complex objects.

Primitive Types

In Java, we have 8 primitive types:

Often, we want to choose a data type that makes sense for our use case. For example, if we want to represent months or the day in a month, we don't need to allocate 8 bytes of memory using the int primitive type (which can represent numbers from -2 Billion to 2 Billion). Instead, we can use the byte privative type (which can represent numbers from -127 to 127):

byte month = 1;
byte dayInMonth = 13;

Let's say we want a counter that represents the population of a city:

int bellinghamPopulation = 97_270;

Perhaps an int would work best in this case as we know there are no cities with a population over 2 billion.
*Note: you can use underscores (_) partition large numbers for readability.*

However, if we want to represent the population of the world, then an int would not be sufficient. So, we would have to use the long primitive type:

long globalPopulation = 8_230_000_000L;

*Note: By default, the Java compiler sees the type of numbers as int, so we must use L at the end of our global population to tell Java that that number is of type long.

We can also store decimal numbers in variables. For example:

double eddingtonNum = 1.57e79; // the num protons in the universe.
double planck = 6.62607015e-34; // Planck's Constant J*s

*Note: Java supports scientific notation for floating-point numbers.*

However, for most use cases, the primitive type double may be overkill, so instead we could use the float primitive type:

float pi = 3.14159F;

*Note: By default, the Java compiler sees the type of decimal numbers as double, so we must use F at the end of the value of pi to tell Java that that number is of type float.

We can also store single characters in variables. For example:

char grade = 'A';

*Note: You must surround chars with single quotes ('). Unlike Python, double quotes (") are reserved for Strings and single quotes are reserved for chars.*

Finally, we can represent Booleans with primitive data-types. For example:

boolean isRaining = true;

Booleans in Java can either be true or false. Unlike Python, Booleans are all lower-case.

Constants

Earlier, I initialized a variable called pi and set it to the value 3.14159. However, the value of pi never changes, and perhaps I don't want someone else to come along and accidentally reassign the value of pi to something else.

In this case, I want to set pi as a constant. We can do this using the final keyword in Java:

final float PI = 3.14159F;

*Note: It is the convention that constant values are all upper-case.**

Now, if I try to reassign PI to another value later, the Java compiler will catch it and throw this error:

RectangleExample.java:13: error: cannot assign a value to final variable PI
        PI = 3F;
        ^
1 error

Reference Types

We use primitive types to represent simple values, but we use reference types to represent complex objects like Strings, Dates, or any other data type beyond the 8 primitive types. For example:

import java.util.Random;

public class RandomTest {
	public static void main(String[] args) {
		Random randomGen = new Random();
	}
}

*Note: I am importing the Random class from the from the java.util package. We will go over imports more later*

In this example, Random is a reference type from the java.util package. When we write Random randomGen = new Random();, we are creating a new Random object and assigning it to the variable randomGen. This variable does not store the object itself but instead stores a reference to the memory location where the object resides.

These objects have members that we can access using the dot (.) operator. For example, I can generate a random integer using my new randomGen object:

Random randomGen = new Random();
int randomInt = randomGen.nextInt();
System.out.println(randomInt);

This could display:

-1308142446

You may be wondering what are these reference types really. For this example, I am going to create an instance of a Rectangle object form the java.awt package:

import java.awt.Rectangle;

public class RectangleExample {
    public static void main(String[] args) {
	    // upper-left point (x,y), its width, and its height.
        Rectangle rect = new Rectangle(10, 20, 50, 100);
        System.out.println(rect.x);
    }
}

Which displays:

10

Now, I will create a new variable rect2 and set it's value to the original rect:

Rectangle rect = new Rectangle(10, 20, 50, 100);
System.out.println(rect.x);
Rectangle rect2 = rect;
System.out.println(rect2.x);

Which displays:

10
10

Currently, it is behaving as we would expect. However, if I change the value of rect.x, how will this affect rect2.x?

Rectangle rect = new Rectangle(10, 20, 50, 100);
System.out.println(rect.x);
Rectangle rect2 = rect;
rect.x = 15;
System.out.println(rect2.x);

This will display:

10
15

Even though I did not change rect2, the values of it's members changed when I changed rect.

Strings

We have seen strings in the first Java program that we wrote:

public class Hello {
	public static void main(String[] args) {
		System.out.println("Hello, World!");
	}
}

This is an example of a string literal However, we can also store this string as a variable of type String:

String hello = new String("Hello, World!");
System.out.println(hello);

Notice that the String type is capitalized and is not one of the eight primitive types. This indicates that Strings are reference types, so we need to instantiate it with the new Java keyword.

However, strings are special in this case, so instead, we can simply write:

String hello = "Hello, World!";
System.out.println(hello);

In Java there are several operations that you can perform on strings. For example, you can concatenate strings together using the addition (+) operator:

String hello = "Hello," + " World!";
System.out.println(hello);

Because String is a reference type, there are several methods we can access with dot-notation. For example:

String message = "Hello, World!";

System.out.println("Length: " + message.length());     System.out.println("Uppercase: " + message.toUpperCase());     System.out.println("Lowercase: " + message.toLowerCase());
System.out.println("Character at index 7: " + message.charAt(7));
System.out.println("Substring (7 to end): " + message.substring(7)); 
System.out.println("Does it contain 'World': "  + message.contains("World"));

Which will output:

Length: 13
Uppercase: HELLO, WORLD!
Lowercase: hello, world!
Character at index 7: W
Substring (7 to end): World!
Does it contain 'World': true

Escape Sequences in Java

Escape sequences are special character combinations starting with a backslash (\) used to represent characters that cannot be typed directly or have a special meaning in strings. They allow you to format strings or include special characters.

Here are the most common escape sequences in Java:

Escape Sequence Description Example Output
\n New line System.out.println("Hello\nWorld"); Hello``World
\t Tab System.out.println("Hello\tWorld"); Hello World
\" Double quote System.out.println("\"Hello\""); "Hello"
\\ Backslash System.out.println("\\"); \
\' Single quote System.out.println("It\'s Java"); It's Java

Command-Line Arguments

Command-line arguments allow you to pass data to a Java program when you execute it from the terminal or command prompt. These arguments are passed as an array of String values to the main method of the program.

The main method signature includes a parameter String[] args, which is used to capture command-line arguments:

public static void main(String[] args) {
    String arg1 = args[0]
    String arg2 = args[1]
    String arg3 = args[2]
    ...
}

Arguments are passed when running the program using the java command. Each argument is separated by a space.

For example:

java Hello arg1 arg2 arg3

arg1, arg2, and arg3 will be stored in the args array as args[0], args[1], and args[2] respectively.

*Quick Aside: To convert a String into an int, you can do this:*

String myNum = "12";
int myInt = Integer.parseInt(myNum); // This will be an int with the value 12.
