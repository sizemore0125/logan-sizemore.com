Arithmetic Expressions

Java has several arithmetic operators that can be used to perform basic mathematical operations. These operators are the same as those used in other programming languages like Python:

Operator Description Example Output
+ Addition 5 + 3 8
- Subtraction 5 - 3 2
* Multiplication 5 * 3 15
/ Division 6 / 3 2
% Modulus (remainder) 7 % 3 1

I don’t expect you to memorize these. You’ll learn them through practice with labs and homework assignments.

Addition and Subtraction

The addition (+) and subtraction (-) operators are used to add and subtract two numbers respectively. Here's an example:

int x = 5;
int y = 10;
int z = x + y;
System.out.println(z);

z = x - y  
System.out.println(z);

This will display:

15
-5

These “behave” just like how you “know” of them from elementary mathematics!

Multiplication and Division

The multiplication (*) and division (/) operators are used to add and subtract two numbers
respectively. Here's an example:

int x = 5; 
int y = 2;
int z = x * y;
System.out.println(z);

z = x / y;
System.out.println(z);

This will display:

10
2

The division operator is also “just like” the elementary math operator you are familiar with, but unlike Python the division operator does not turn z into a float. Instead, z remains and integer and result is rounded down.

Modulo

The modulo operator (%) is used to get the remainder of a division operation. For example:

int z = 10 % 2;
System.out.println(z);

z = 7 % 1;
System.out.println(z);

z = 67 % 14;
System.out.println(z);

This will display:

1
0
11

Augmented Assignment

The assignment operator (=) can also be combined with arithmetic operators to perform arithmetic operations and assign the result to a variable. This is known as augmented assignment.

Augmented assignment is used to perform an operation and assign the result back to the
variable. Here are some examples:

// Example 1: Addition  
int x = 5  
x += 3  // Equivalent to x = x + 3  
System.out.println(x);

// Example 2: Subtraction  
x = 5  
x -= 3 // Equivalent to x = x - 3  
System.out.println(x);

// Example 3: Multiplication  
x = 5  
x *= 3 // Equivalent to x = x * 3  
System.out.println(x);

// Example 4: Division  
x = 10  
x /= 2  
// Equivalent to x = x / 2  
System.out.println(x);

// Example 5: Modulo  
x = 10  
x %= 3 // Equivalent to x = x % 3  
System.out.println(x);

This will display:

8
2
15
5
1

In addition to the Augmented Assignment operators defined above, we have the increment (++) and decrement (--) operators. These work similarly to the addition and subtraction augmented assignment operators.

For example:

int x = 1;
x++; // Equivalent to x += 1 and x = x + 1
System.out.println(x);

This will display:

2

We can use the increment (or decrement) operator as both a postfix (x++) and a prefix (++x). These two versions generally work the same way. However, there is a subtlety when using them in assignment.

For example:

int x = 1;
int y = x++;
System.out.println(x);
System.out.println(y);

x = 1;
y = ++x;
System.out.println(x);
System.out.println(y);

This will display:

2
1
2
2

When the increment operator is used as a postfix, the value of x is assigned to y before x is incremented. When the increment operator is used as a prefix, the value of x is assigned to y after x is incremented.

Order of Operations

In Java, just like in mathematics, there is an order of operations that determines the sequence in which arithmetic operations are performed. The order of operations is important to understand, as it can affect the result of an expression.

The order of operations in Java is as follows:

  • Parentheses
  • Multiplication and Division (from left to right)
  • Addition and Subtraction (from left to right)

This means that any expressions inside parentheses are evaluated first, followed any multiplication or division operations, and finally any addition or subtraction operations.

Casting

In Java, casting allows us to explicitly or implicitly convert a value from one type to another. Casting is necessary when working with data types that differ in size or precision.

There are two types of Casting: Implicit Casting and Explicit Casting

Implicit Casting (widening)

This is performed automatically by the compiler when converting a smaller type to a larger type.

For example:

short x = 10;
int y = x + 1;
System.out.println(y)

This will display:

11

The variable x was converted automatically from a short to a int. Implicit casting is safe because there is no loss of data.

Implicit casting can occur when the target type has a larger range or higher precision than the source type. For example: byte → short → int → long → float → double

Explicit Casting (narrowing)

Explicit casting must be performed manually when converting a larger type to a smaller type. This is done by placing the target type in parentheses before the value.

If we attempt:

int x = 10;
short y = x + 1;
System.out.println(y)

It will result in this error:

Main.java:4: error: incompatible types: possible lossy conversion from int to short
    short y = x + 1;
                ^
1 error

Instead, we need to explicitly cast x to type short in order for our expression to work. We do this by prefixing x with the type we want x to become in paratheses (short).

For example:

int x = 10;
short y = (short)x + 1;
System.out.println(y)

This will display:

11

String Casting

So far, we have seen how we can change primitive types into other primitive types. However, what if we want to turn a variable of type String into an int. This is especially important when parsing user input, as it often comes in the form of a String.

Each primitive type has an associated wrapper class (e.g., Integer, Double, etc.) with a static static method that takes the form parseXxx(), which can be used to convert a String to the corresponding primitive type.

Method Target Type Example Output
Integer.parseInt() int Integer.parseInt("123") 123
Double.parseDouble() double Double.parseDouble("3.14") 3.14
Boolean.parseBoolean() boolean Boolean.parseBoolean("true") true

For example:

public class Hello {
	public static void main(String[] args) {
		String userString = args[0];
		int userInt = Integer.parseInt(userString) + 10;
		System.out.println(userInt);
	}
}

Executed like this:

java Hello 21

This will display:

31

Comparison (Relational) Operators

Java also provides several comparison operators for comparing Boolean values. The most
common comparison operators are:

Operator Description Example Output
== Equal to 5 == 5 true
!= Not equal to 5 != 3 true
> Greater than 5 > 3 true
< Less than 5 < 3 false
>= Greater than or equal to 5 >= 5 true
<= Less than or equal to 5 <= 3 false

For example:

int x = 10;
System.out.println(x == 11);

This will display:

false

*Note: The assignment operator (=) is very different from the equality operator (==).*

Assignment Operator (=): You can think of this as a command. int x = 5; means "x IS 5". Equality Operator (==): You can think of this as a question. x == 5; means "is x equal to 5?"

Other Examples:

int x = 10;
System.out.println(x != 11);
System.out.println(x > 10);
System.out.println(x >= 10);
System.out.println(x < 10);
System.out.println(x <= 10);

This will display:

true
false
true
false
true

String Comparison

In Java, you cannot use the equality operator (==) to compare strings. When you use the equality operator on reference types, the memory location is compared, not any other value associated with that object. So, instead objects of type String are compared using specific methods to ensure correctness.

The .equals() method checks whether the content of two strings is the same. For example:

String str1 = "Java"; 
String str2 = "Java";  

if (str1.equals(str2)) {     
	System.out.println("The strings are equal."); 
} else {     
	System.out.println("The strings are not equal."); 
}

This will display:

The strings are equal.

*Note: You can also use .equalsIgnoreCase() if you want to ignore any case differences between two strings.*

Logical Operators

Java provides several logical operators for working with Boolean values. The most
common logical operators are:

Operator Description Example Output
&& Logical AND true && false false
\|\| Logical OR true \|\| false true
! Logical NOT !true false

For example:

boolean isRaining = true;
boolean hasUmbrella = false

boolean goOutside = !isRaining || (isRaining && hasUmbrella) // Output: false

*Note: This Boolean expression can be simplified to !isRaining || hasUmbrella*
